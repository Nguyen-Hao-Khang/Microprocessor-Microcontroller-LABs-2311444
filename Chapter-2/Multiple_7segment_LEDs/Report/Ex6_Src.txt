////////////////////////////////////////////////////////////////////////////////////////////////////
/////////////////////////////////// REPORT 1 ///////////////////////////////////////////////
////////////////////////////////////////////////////////////////////////////////////////////////////

If the line setTimer0(1000); is missing, the software timer won’t be initialized properly. As a result, the timer0_counter remains at zero, and the timer0_flag will never be set. This means the condition if(timer0_flag == 1) in the main loop will never be true, so the LED won’t blink at all. In short, without initializing the timer, the system has no way to track time intervals, and the LED control logic fails to execute.





////////////////////////////////////////////////////////////////////////////////////////////////////
/////////////////////////////////// REPORT 2 ///////////////////////////////////////////////
////////////////////////////////////////////////////////////////////////////////////////////////////

Changing the line to setTimer0(1); causes a small but serious problem. Since the timer uses integer division, 1 / TIMER_CYCLE (assuming TIMER_CYCLE = 10) will return 0. This means that timer0_counter is set to 0 immediately. The result of this change is the same as in the previous statement, the LED will not blink as expected.







////////////////////////////////////////////////////////////////////////////////////////////////////
/////////////////////////////////// REPORT 3 ///////////////////////////////////////////////
////////////////////////////////////////////////////////////////////////////////////////////////////

If the command line is changed to setTimer0(10); then timer0_counter = 10 / 10 = 1. This means that if (timer_flag[0] == 1) is executed almost immediately, the light will change state. The next state changes will be executed every 2s due to the setTimer0(2000) statement. Unlike the previous 2 cases, the light will blink after adjusting timer0_counter >= 1 to make the counter active and set the signal flag in the main loop.
